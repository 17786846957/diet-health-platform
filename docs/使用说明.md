# 智能饮食健康管理平台 — 使用说明

## 一、环境要求

| 软件 | 最低版本 | 推荐版本 | 说明 |

## 二、IDE 配置与项目展示

### 方式一：IntelliJ IDEA + VS Code（推荐）

两个 IDE 各司其职，体验最佳。

#### IDEA 打开后端

1. **File → Open** → 选择 `D:/bishe2/diet-health-backend`（含 `pom.xml` 的目录）
2. IDEA 自动识别 Maven 项目，右下角弹出提示时点击 **Import Changes**，等待依赖下载完成
3. 在项目树中找到主启动类（带 `@SpringBootApplication` 注解），右键 **Run**
4. 控制台出现 `Started Application in x.xx seconds` 表示启动成功
5. 后端运行在 http://localhost:8082/api

**常用操作：**
| 操作 | 方法 |
|------|------|
| 启动/停止 | 右上角绿色三角 ▶ / 红色方块 ■ |
| 查看端口 | `application.yml` 中 `server.port` |
| 查看日志 | 底部 Run 面板 |
| 热重载 | 修改代码后点锤子图标 🔨 或快捷键 Ctrl+F9 |

#### VS Code 打开前端

1. 打开 VS Code，**File → Open Folder** → 选择 `D:/bishe2/diet-health-frontend`
2. 按 `` Ctrl+` `` 打开终端，运行：
   ```bash
   npm install    # 首次需要安装依赖
   npm run dev    # 启动 Vite 开发服务器
   ```
3. 浏览器访问终端输出的地址（通常是 http://localhost:5173）
4. 修改 `.vue` 文件后页面自动热更新

**推荐安装的扩展（Ctrl+Shift+X 搜索安装）：**
| 扩展名 | 用途 |
|--------|------|
| Vue - Official | Vue 3 语法高亮、智能提示 |
| ESLint | 代码规范检查 |
| Prettier | 代码格式化 |
| Auto Close Tag | 自动闭合 HTML 标签 |

## 三、安装步骤

### 方式一：本地开发启动

#### 1. 数据库初始化

```bash
# 登录 MySQL，执行建表和初始数据脚本
mysql -u root -p < sql/init.sql
mysql -u root -p diet_health < sql/food_expand.sql
```

执行后会创建 `diet_health` 数据库，包含以下核心表：

| 表名 | 说明 | 初始数据 |
|------|------|---------|
| user | 系统用户 | admin 管理员账号 |
| food | 食物库 | 230+ 种中式食物 |
| diet_record | 饮食记录主表 | — |
| diet_record_detail | 饮食记录明细 | — |
| family_member | 家庭成员 | — |
| food_favorite | 食物收藏 | — |
| health_goal | 健康目标 | — |
| water_record | 饮水记录 | — |
| exercise_record | 运动记录 | — |
| weight_record | 体重记录 | — |
| body_symptom | 身体症状 | — |

#### 2. 配置后端

编辑 `diet-health-backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/diet_health?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
    username: root
    password: your_password  # 改为你的 MySQL 密码

jwt:
  secret: your-secret-key    # JWT 签名密钥，至少 32 字符
```

#### 3. 启动后端

```bash
cd diet-health-backend
mvn spring-boot:run
```

后端运行在 http://localhost:8082/api。

验证后端启动成功：
```bash
curl http://localhost:8082/api/food/list
# 应返回食物列表 JSON
```

#### 4. 启动前端

```bash
cd diet-health-frontend
npm install
npm run dev
```

前端运行在 http://localhost:5173。

### 方式二：Docker 一键部署

```bash
# 可选：设置 JWT 密钥
export JWT_SECRET=$(openssl rand -base64 32)

# 启动所有服务（MySQL + 后端 + Nginx 前端）
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f backend
```

| 服务 | 容器名 | 端口 | 说明 |
|------|--------|------|------|
| MySQL | diet-mysql | 3306 | 数据库，healthcheck 就绪后才启动后端 |
| 后端 | diet-backend | 8082 | Spring Boot API 服务 |
| 前端 | diet-frontend | 80 | Nginx 静态文件 + API 反向代理 |

访问 http://localhost 即可使用系统。

## 四、功能使用

### 4.1 注册与登录

1. 访问系统，自动跳转登录页
2. 点击"注册"创建账号
   - 用户名：2-20 个字符
   - 密码：6-50 位，需包含字母和数字
3. 注册后自动跳转登录页，输入账号密码登录
4. 登录成功后跳转首页仪表盘

### 4.2 首页仪表盘

首页展示四个统计卡片（今日热量/蛋白质/脂肪/碳水），每个有进度条显示完成度。下方有热量趋势折线图（支持 7/14/30 天切换）和快捷操作按钮。

### 4.3 饮食记录

1. 进入"饮食记录"页面
2. 顶部选择日期（默认今天）
3. 点击"新增记录"按钮
4. 选择餐次：早餐 / 午餐 / 晚餐 / 加餐
5. 搜索食物（支持中文名称和拼音首字母，如输入 "jt" 搜索 "鸡腿"）
6. 选择食物后设置食用量（单位：克）
7. 可添加多种食物，点击"保存"
8. 系统自动计算热量 = 食物热量(kcal/100g) × 食用量(g) / 100

**热量预警：** 当日热量达到目标 80% 显示橙色警告，达到 100% 显示红色超标提示。

### 4.4 食物库

1. 进入"食物库"页面
2. 支持按分类筛选（主食/蔬菜/水果/肉类/蛋奶/零食/饮品/家常菜/调味品/外卖/收藏）
3. 支持关键词搜索（中文名称、拼音首字母）
4. 点击食物查看详细营养信息（热量、蛋白质、脂肪、碳水、膳食纤维）
5. 可收藏常用食物

### 4.5 营养统计

| 图表 | 说明 |
|------|------|
| 热量趋势折线图 | 近 7 / 14 / 30 天的每日热量变化，带目标热量标线 |
| 营养素占比饼图 | 蛋白质、脂肪、碳水的摄入占比环形图 |
| 目标完成度柱状图 | 各营养素的完成百分比横向条形图 |
| 每日营养明细 | 按餐次分组展示食物及热量 |

### 4.6 智能饮食建议

进入"智能饮食建议"页面，包含三个 Tab：

**每日建议：** 显示当前营养状态（已达标/接近达标/未达标）、营养缺口分析、智能推荐食物（按餐次切换，含推荐理由和推荐分）。

**饮食分析：** 统计周期内平均每日摄入、食物种类数、餐次分布饼图。

**健康建议：** 根据用户目标（减脂/增肌/维持）给出个性化建议。

### 4.7 饮水记录

1. 进入"饮水记录"页面
2. 快捷添加按钮：150 / 200 / 250 / 300 / 500 ml
3. 支持选择饮品类型：水 / 茶 / 咖啡 / 果汁 / 牛奶 / 其他
4. 今日饮水量进度条（目标 2000ml）
5. 本周饮水趋势柱状图

### 4.8 运动记录

1. 进入"运动记录"页面
2. 常见运动快捷按钮：跑步/快走/游泳/骑行/瑜伽/健身/跳绳/球类
3. 填写运动时长（分钟），系统基于 MET 值自动计算热量消耗
4. 选择强度：低 / 中 / 高
5. 本周运动趋势图（时长 + 消耗双柱）

### 4.9 体重记录

1. 进入"体重记录"页面
2. 输入体重(kg)和体脂率(%)
3. 查看体重 + 体脂率双 Y 轴折线图
4. BMI 参考值和等级（偏瘦/正常/偏胖/肥胖）
5. 数据统计：起始体重/最新体重/体重变化/记录次数

### 4.10 家庭成员管理

1. 进入"个人中心"页面
2. 在"家庭成员管理"区域点击"添加成员"
3. 填写成员信息：姓名、性别、年龄、身高(cm)、体重(kg)、活动水平、健康目标
4. 顶部导航栏可切换成员视角
5. 切换后，饮食记录、营养统计、运动记录等自动过滤为对应成员的数据

**数据隔离机制：** 每条记录通过 `member_id` 字段关联到具体成员（NULL 表示主用户），查询时自动过滤。

### 4.11 管理员后台

使用管理员账号登录后，进入"后台管理"页面：

| 功能 | 说明 |
|------|------|
| 仪表盘 | 用户总数、食物总数、记录总数统计 |
| 用户管理 | 查看用户列表、搜索、删除用户 |
| 食物管理 | 添加、编辑、删除食物数据 |

**食物数据管理：** 对食物库进行增删改操作（普通用户只能查询/浏览），包括添加新食物、编辑营养信息、删除食物记录。

**系统数据总览：** 查看平台级统计——注册用户数、食物种类数、饮食记录总数。

**API 文档访问：** 访问 Swagger UI（`/swagger-ui.html`），普通用户无法查看。

### 5.4 权限控制实现机制

系统采用三层防线确保权限安全：

| 层级 | 机制 | 说明 |
|------|------|------|
| 数据库 | `user` 表 `role` 字段 | 默认 `'user'`，注册时硬编码，无法通过接口提权 |
| 认证层 | JWT Token 嵌入角色 | 登录时 role 写入 Token，Filter 解析后构造 `ROLE_admin` 权限 |
| 授权层 | SecurityConfig URL 拦截 | `/admin/**` 和 Swagger 路径要求 `hasRole('admin')` |
| 前端 | 路由守卫 + 菜单隐藏 | `requiresAdmin` 守卫重定向 + 侧边栏 `v-if` 隐藏入口 |

### 5.5 管理后台使用

1. 使用 admin 账号登录系统
2. 侧边栏出现"管理后台"菜单项（普通用户不可见）
3. 点击进入管理后台，包含两个 Tab：
   - **用户管理**：仪表盘统计卡片 + 用户列表表格（支持搜索、分页、删除）
   - **食物管理**：搜索筛选 + 食物列表（支持新增、编辑、删除）

## 七、常见问题

### Q: 登录后页面空白？

**A:** 检查以下几点：
1. 后端是否正常启动（`curl http://localhost:8082/api/food/list` 是否返回 JSON）
2. MySQL 数据库是否已初始化（`mysql -u root -p -e "USE diet_health; SHOW TABLES;"`）
3. 浏览器控制台是否有报错（F12 → Console）

### Q: 食物搜索没有结果？

**A:** 系统内置 230+ 种中式食物数据，搜索支持中文名称和拼音首字母。如果搜索无结果，可能是关键词不在食物库中，可通过管理员后台添加新食物。

### Q: Docker 部署失败？

**A:** 常见原因：
1. Docker 或 Docker Compose 未安装 → 安装后重试
2. 端口被占用 → 检查 80、8082、3306 端口：`netstat -tlnp | grep -E '80|8082|3306'`
3. MySQL 启动慢 → 等待 healthcheck 通过：`docker-compose ps` 查看状态
4. 内存不足 → Docker 至少分配 2GB 内存

### Q: 忘记管理员密码？

**A:** 直接执行 SQL 重置，或重新导入 init.sql。

### Q: 如何修改端口？

**A:**
- 后端端口：修改 `application.yml` 中 `server.port`
- 前端端口：修改 `vite.config.js` 中 `server.port`
- Docker 端口：修改 `docker-compose.yml` 中 `ports` 映射

### Q: 如何备份数据？

**A:**
```bash
# 本地 MySQL
mysqldump -u root -p diet_health > backup.sql

# Docker MySQL
docker exec diet-mysql mysqldump -u root -p123456 diet_health > backup.sql
```
