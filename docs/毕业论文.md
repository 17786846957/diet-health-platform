# 基于 Spring Boot 的智能饮食健康管理平台的设计与实现

## 摘要

随着"健康中国2030"战略的推进，个人及家庭饮食健康管理需求日益增长。现有饮食管理工具大多面向单用户设计，无法满足家庭场景下多成员独立管理的需求。针对这一问题，本文设计并实现了一个基于 Spring Boot 的家庭饮食健康管理平台。

系统采用前后端分离架构，后端基于 Spring Boot 2.7 + MyBatis-Plus + Spring Security，前端采用 Vue 3 + Element Plus + ECharts，数据库选用 MySQL 8.0，通过 Docker Compose 实现一键部署。

系统实现了用户认证、饮食记录管理、食物库检索、营养统计分析、家庭成员管理、个性化饮食推荐、饮水记录、运动记录、体重管理和健康目标等核心功能。通过 member_id 字段实现家庭成员数据隔离，通过六维度加权评分推荐算法提供差异化饮食建议。

测试结果表明，系统包含 202 个后端测试和 53 个 E2E 测试，全部通过。系统功能完整、运行稳定，能够有效满足家庭场景下的饮食健康管理需求。

**关键词：** Spring Boot；Vue 3；饮食健康管理；家庭成员数据隔离；推荐算法

## Abstract

With the advancement of the "Healthy China 2030" strategy, the demand for personal and family diet health management is growing. Most existing diet management tools are designed for single users and cannot meet the needs of multi-member independent management in family scenarios. This paper designs and implements a family diet health management platform based on Spring Boot.

The system adopts a front-end and back-end separation architecture. The back-end is based on Spring Boot 2.7 + MyBatis-Plus + Spring Security, and the front-end uses Vue 3 + Element Plus + ECharts. MySQL 8.0 is used as the database, and one-click deployment is achieved through Docker Compose.

The system implements core functions including user authentication, diet record management, food library search, nutrition statistics analysis, family member management, personalized diet recommendation, water intake tracking, exercise recording, weight management, and health goal setting. Family member data isolation is realized through the member_id field, and differentiated dietary suggestions are provided through a six-dimensional weighted scoring recommendation algorithm.

Test results show that the system contains 202 backend tests and 53 E2E tests, all of which pass. The system has complete functions and stable operation, which can effectively meet the needs of diet health management in family scenarios.

**Keywords:** Spring Boot; Vue 3; Diet Health Management; Family Member Data Isolation; Recommendation Algorithm

## 第1章 绪论

### 1.1 选题背景与意义

随着我国经济社会的快速发展和人民生活水平的不断提高，饮食健康问题日益受到社会关注。2016年，国务院发布《"健康中国2030"规划纲要》，明确提出要建立健全健康教育体系。2019年，国务院印发《健康中国行动（2019—2030年）》，将合理膳食行动列为15项重大行动之一。

在这一政策背景下，饮食健康管理类应用应运而生。然而，现有工具普遍存在以下不足：第一，大多面向单用户设计，不支持家庭成员间的独立数据管理；第二，营养推荐算法较为简单，缺乏对用户历史偏好和营养缺口的综合考量；第三，部分系统采用传统单体架构，前后端耦合度高，不利于维护和扩展。

家庭是社会的基本单位，家庭成员的饮食健康相互影响。一个能够支持多成员独立管理饮食数据的平台，不仅有助于每个家庭成员了解自身的营养摄入状况，还能为家庭整体的饮食规划提供数据支撑。因此，设计并实现一个支持家庭成员数据隔离的饮食健康管理平台具有重要的现实意义。

### 1.2 国内外研究现状

**国外方面**，MyFitnessPal、Lose It! 等应用在全球范围内拥有大量用户。MyFitnessPal 提供了庞大的食物数据库和热量追踪功能，但家庭管理功能有限。Lose It! 在个性化推荐方面有所探索，但推荐算法主要基于用户设定的目标，而非动态分析实际营养缺口。

**国内方面**，薄荷健康、Keep 等平台在饮食管理领域具有较高的市场占有率。薄荷健康提供了较为完善的食物营养数据库，但缺乏家庭成员维度的数据隔离设计。Keep 侧重于运动健身，饮食管理功能相对薄弱。

在技术架构方面，Spring Boot + Vue 的技术组合已成为 Web 应用开发的主流选择。Spring Boot 以其快速开发和丰富的生态系统成为 Java 后端的主流框架；Vue 3 以其响应式数据绑定和组件化开发成为前端的热门选择。

### 1.3 主要研究内容

本文的主要研究内容包括：

（1）分析家庭饮食健康管理需求，设计系统功能模块和数据库结构。

（2）实现基于 Spring Boot 的后端服务，包括用户认证、饮食记录、食物库管理、营养统计、家庭成员管理、饮食推荐、饮水记录、运动记录、体重管理、健康目标和身体症状等 13 个功能模块。

（3）实现基于 Vue 3 的前端应用，包括 14 个页面组件、数据可视化图表和响应式布局。

（4）设计并实现家庭成员数据隔离机制，通过 member_id 字段实现同一账号下多成员的独立数据管理。

（5）设计并实现基于六维度加权评分的饮食推荐算法，综合考虑营养适配度、餐次适配度、食物健康度、历史偏好、收藏行为和分类多样性。

（6）实现 Docker 容器化部署方案，支持一键启动。

## 第2章 相关技术介绍

### 2.1 Spring Boot 框架

Spring Boot 是基于 Spring 框架的快速开发脚手架，通过自动配置机制简化应用创建过程。本系统使用 Spring Boot 2.7.18 版本，基于 Spring Framework 5.3，支持 Java 11 及以上版本。主要特性包括：自动配置、起步依赖、内嵌 Web 服务器、Actuator 监控端点等。

### 2.2 MyBatis-Plus

MyBatis-Plus 是 MyBatis 的增强工具，提供了通用 CRUD 操作接口、条件构造器、分页插件等功能。本系统使用 3.5.5 版本，通过 BaseMapper 获得通用 CRUD 方法，使用 LambdaQueryWrapper 构建类型安全的查询条件。

### 2.3 Spring Security 与 JWT

Spring Security 是 Spring 生态中的安全框架，提供认证、授权和攻击防护。JWT（JSON Web Token）是开放标准（RFC 7519），用于安全传输信息。本系统将用户 ID、用户名和角色编码到 Token 中，通过 httpOnly Cookie 传输，有效防止 XSS 攻击。

### 2.4 Vue 3 框架

Vue 3 是 Vue.js 的最新主要版本，引入了 Composition API、Teleport、Suspense 等新特性。Composition API 允许使用函数的方式组织组件逻辑，通过 ref、reactive、computed、watch 等函数实现响应式数据管理，在逻辑复用和代码组织方面具有明显优势。

### 2.5 Element Plus 与 ECharts

Element Plus 是专为 Vue 3 设计的桌面端 UI 组件库，提供了按钮、表单、表格、对话框等丰富组件。ECharts 是百度开源的数据可视化库，提供了折线图、柱状图、饼图等丰富的图表类型。

### 2.6 Docker 与 Docker Compose

Docker 是开源的容器化平台，允许将应用及其依赖打包到轻量级、可移植的容器中。Docker Compose 通过 YAML 文件定义和运行多容器应用。本系统使用 Docker Compose 编排 MySQL、后端和前端三个服务，实现一键部署。

## 第3章 系统需求分析

### 3.1 功能需求分析

#### 3.1.1 用户管理模块

（1）用户注册：输入用户名和密码完成注册，密码使用 BCrypt 加密存储。

（2）用户登录：验证身份后生成 JWT Token，通过 httpOnly Cookie 返回。

（3）个人信息管理：查看和修改性别、年龄、身高、体重、活动水平和健康目标。

（4）退出登录：清除 Cookie 中的 Token。

#### 3.1.2 饮食记录管理模块

（1）添加饮食记录：选择日期和餐次（早餐、午餐、晚餐、加餐），搜索食物并设置食用量。

（2）编辑/删除记录：修改或删除已有记录。

（3）按日期查询：查看当日饮食记录列表，显示餐次、食物明细和总热量。

（4）热量预警：当日热量接近或超过目标时显示预警。

#### 3.1.3 食物库管理模块

（1）食物列表：分页展示，支持分类筛选和关键词搜索（含拼音首字母匹配）。

（2）食物收藏：用户可收藏常用食物，收藏列表单独展示。

（3）食物管理（管理员）：添加、编辑和删除食物数据。

#### 3.1.4 营养统计分析模块

（1）每日统计：展示当日热量、蛋白质、脂肪和碳水摄入总量。

（2）热量趋势：折线图展示近 7/14/30 天的每日热量摄入。

（3）营养素占比：饼图展示蛋白质、脂肪和碳水的摄入占比。

（4）营养缺口分析：对比实际摄入与目标值。

#### 3.1.5 家庭成员管理模块

（1）成员列表：展示所有家庭成员的健康信息。

（2）添加/编辑/删除成员：管理家庭成员档案。

（3）成员切换：切换后饮食记录、营养统计自动过滤为对应成员数据。

#### 3.1.6 饮食推荐模块

（1）智能推荐：根据当前餐次和营养缺口推荐食物。

（2）推荐理由：每条推荐附带说明。

（3）快速添加：点击推荐食物直接打开新增记录对话框。

#### 3.1.7 后台管理模块

（1）仪表盘：展示用户总数、食物总数和记录总数。

（2）用户管理：查看、搜索、删除用户。

### 3.2 非功能需求分析

#### 3.2.1 安全性需求

- 认证安全：JWT httpOnly Cookie 防 XSS
- 密码安全：BCrypt 加密存储
- 接口安全：角色权限控制
- 防暴力破解：登录限流 5 次/分钟

#### 3.2.2 可靠性需求

- 数据一致性：定时校验饮食记录总热量
- 全局异常处理：统一异常处理机制

#### 3.2.3 可维护性需求

- API 文档：Swagger 自动生成
- 日志系统：SLF4J + Logback
- 容器化部署：Docker Compose 一键部署

## 第4章 系统设计

### 4.1 系统总体架构设计

系统采用前后端分离的 B/S 架构，分为三个层次：

```
┌─────────────────────────────────────────────────────────┐
│                      表现层 (Presentation)                │
│  ┌─────────┐ ┌──────────┐ ┌────────┐ ┌───────────────┐  │

│  │  框架   │ │  Plus    │ │ 可视化 │ │  状态管理     │  │
│  └─────────┘ └──────────┘ └────────┘ └───────────────┘  │
│           ↕ HTTP/REST API (JSON)                         │
├─────────────────────────────────────────────────────────┤
│                      业务层 (Business)                    │
│  ┌──────────────┐ ┌────────────┐ ┌───────────────────┐  │
│  │ Spring Boot  │ │  Spring    │ │   MyBatis-Plus    │  │
│  │   2.7        │ │  Security  │ │   ORM 框架        │  │
│  └──────────────┘ └────────────┘ └───────────────────┘  │
│           ↕ JDBC                                         │
├─────────────────────────────────────────────────────────┤
│                      数据层 (Data)                        │
│              ┌──────────────────────┐                    │
│              │    MySQL 8.0         │                    │
│              │  用户/食物/记录/成员  │                    │
│              └──────────────────────┘                    │
└─────────────────────────────────────────────────────────┘
```

**表现层**：基于 Vue 3 的单页应用，使用 Element Plus 作为 UI 组件库，ECharts 实现数据可视化，Pinia 进行状态管理。前端通过 axios 封装 HTTP 请求，请求拦截器自动携带 JWT Token，响应拦截器统一处理 401 和业务异常。

**业务层**：基于 Spring Boot 2.7 的 RESTful API 服务，使用 Spring Security 实现认证授权，MyBatis-Plus 实现数据持久化。Controller 层负责参数校验和路由，Service 层承载业务逻辑，Mapper 层负责数据库操作。

**数据层**：MySQL 8.0 关系型数据库，包含 11 张业务表，通过外键关联保证数据一致性。

### 4.2 系统数据流设计

以"添加饮食记录"为例，展示系统的完整数据流：

```
用户操作 → 前端表单 → axios API 调用 → 后端 Controller 参数校验
    → Service 业务逻辑（计算热量、校验归属）→ Mapper 数据库操作
    → 返回 R 对象 → 前端更新页面
```

关键数据流说明：

（1）**认证流程**：登录请求 → AuthController 验证密码 → JwtUtil 生成 Token → Set-Cookie 响应头写入 httpOnly Cookie → 后续请求 JwtAuthenticationFilter 从 Cookie 提取 Token 解析用户信息。

（2）**饮食记录流程**：前端提交 JSON（日期 + 餐次 + 食物明细列表）→ Service 遍历明细计算热量（食物热量 × 食用量 / 100）→ 汇总总热量 → 插入 diet_record 和 diet_record_detail 两张表。

（3）**推荐流程**：前端请求推荐 → Service 查询当日已摄入数据 → 计算营养缺口 → SQL 预筛选相关食物 → 多维度加权评分 → 返回 Top 10。

### 5.1 用户认证模块实现

#### 5.1.1 注册功能

用户注册通过 AuthController 的 register 方法实现。校验用户名唯一性，使用 BCrypt 加密密码，插入用户记录。

```java
@Transactional(rollbackFor = Exception.class)
public void register(User user) {
    LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(User::getUsername, user.getUsername());
    if (userMapper.selectCount(wrapper) > 0) {
        throw new BusinessException(ResultCode.USER_EXISTS);
    }
    user.setPassword(passwordEncoder.encode(user.getPassword()));
    user.setRole("user");
    userMapper.insert(user);
}
```

#### 5.1.2 登录与 JWT 认证

登录验证用户名密码后生成 JWT Token，通过 httpOnly Cookie 返回。JWT 认证过滤器 JwtAuthenticationFilter 在每次请求时从 Cookie 或 Authorization Header 提取 Token，解析后创建认证对象放入 SecurityContext。Token 剩余不足 2 小时时自动续期。

```java
private String resolveToken(HttpServletRequest request) {
    Cookie cookie = WebUtils.getCookie(request, COOKIE_NAME);
    if (cookie != null && !cookie.getValue().isEmpty()) {
        return cookie.getValue();
    }
    String authHeader = request.getHeader("Authorization");
    if (authHeader != null && authHeader.startsWith("Bearer ")) {
        return authHeader.substring(7);
    }
    return null;
}
```

Cookie 安全设置通过 Set-Cookie 响应头手动实现：

```java
public static void addTokenCookie(HttpServletRequest request,
                                   HttpServletResponse response, String token) {
    String secure = request.isSecure() ? "; Secure" : "";
    response.addHeader("Set-Cookie",
        COOKIE_NAME + "=" + token
            + "; Path=/; HttpOnly; Max-Age=86400; SameSite=Lax" + secure);
}
```

### 5.2 饮食记录模块实现

#### 5.2.1 添加饮食记录

前端提交包含日期、餐次和食物明细的 JSON 数据，后端自动计算每条明细的热量（食物热量×食用量/100），汇总为记录总热量后插入数据库。

```java
@Transactional(rollbackFor = Exception.class)
public void addRecord(DietRecord record) {

    }
}
```

### 5.4 家庭成员管理模块实现

#### 5.4.1 数据隔离机制

通过 diet_record 表的 member_id 字段实现数据隔离：

```java
private LambdaQueryWrapper<DietRecord> userMemberWrapper(Long userId, Long memberId) {
    LambdaQueryWrapper<DietRecord> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(DietRecord::getUserId, userId);
    if (memberId == null) {
        wrapper.isNull(DietRecord::getMemberId);
    } else {
        wrapper.eq(DietRecord::getMemberId, memberId);
    }
    return wrapper;
}
```

所有查询方法均使用此辅助方法构建条件，确保只返回当前成员的数据。

### 5.5 饮食推荐算法实现

#### 5.5.1 算法设计

推荐算法基于六维度加权评分策略：

（1）营养适配度（30%）：评估食物能否填补当前蛋白质/脂肪/碳水缺口，计算食物营养成分与用户缺口的匹配程度。

（2）餐次适配度（25%）：不同餐次适合不同类型的食物，早餐适合主食/蛋奶，午晚餐适合肉类/蔬菜。

（3）食物健康度（20%）：高蛋白低热量的食物加分，纯糖/高脂食物惩罚。

（4）历史偏好（15%）：统计近 30 天消费频次，常吃的食物得分更高。

（5）收藏偏好（5%）：收藏过的食物加分。

（6）分类多样性（5%）：鼓励尝试不同类别的食物。

```java
double totalScore = nutritionScore * 0.30
        + mealFitScore * 0.25
        + healthScore * 0.20
        + historyScore * 0.15
        + favoriteScore * 0.05
        + diversityScore * 0.05;
```

#### 5.5.2 推荐流程

首先查询用户当日已摄入的营养数据，计算蛋白质、脂肪、碳水的缺口值。然后从食物库中获取候选食物，对每种食物计算六个维度的评分，加权求和后排序返回 Top 10。每条推荐附带推荐理由文案（如"高蛋白""适合补充当前营养缺口""常吃的食物"等）。

#### 5.5.3 营养目标计算

使用 Harris-Benedict 公式计算 BMR，乘以活动系数和目标调整值得到 TDEE。

- 男性：BMR = 88.362 + 13.397×体重 + 4.799×身高 - 5.677×年龄
- 女性：BMR = 447.593 + 9.247×体重 + 3.098×身高 - 4.330×年龄

活动系数：久坐 1.2、轻度 1.375、中度 1.55、活跃 1.725、极活跃 1.9

目标调整：减脂 -500kcal、维持 0、增肌 +300kcal

### 5.6 全局异常处理实现

GlobalExceptionHandler 统一处理各类异常：BusinessException 映射 HTTP 状态码、MethodArgumentNotValidException 返回 400、BadCredentialsException 返回 401、AccessDeniedException 返回 403、未知 Exception 返回 500。

### 5.7 Docker 部署实现

docker-compose.yml 编排三个服务：MySQL 8.0（healthcheck 确保就绪）、Spring Boot 后端（依赖 MySQL）、Nginx 前端（静态文件 + API 反向代理）。一条 `docker-compose up -d` 命令完成部署。

```
│  │              │    │              │    │            │ │
```

启动顺序通过 depends_on + healthcheck 控制：MySQL healthcheck 通过后才启动后端，后端启动完成后才启动前端。MySQL 数据通过 Docker Volume 持久化，容器重建不丢失数据。

## 第7章 总结与展望

### 7.1 工作总结

本文设计并实现了一个基于 Spring Boot 的家庭饮食健康管理平台，主要完成了以下工作：

（1）采用前后端分离 B/S 架构，后端 Spring Boot + MyBatis-Plus，前端 Vue 3 + Element Plus，MySQL 8.0 数据库，Docker Compose 容器化部署。

（2）实现了用户认证、饮食记录、食物库管理、营养统计、家庭成员管理、饮食推荐、饮食建议、饮水记录、运动记录、体重管理、健康目标、身体症状和后台管理等 13 个功能模块，共 50 余个 RESTful API 接口。

（3）设计并实现了家庭成员数据隔离机制，通过 member_id 字段实现同一账号下多成员的独立数据管理。

（4）设计并实现了基于六维度加权评分的饮食推荐算法，综合营养适配度（30%）、餐次适配度（25%）、食物健康度（20%）、历史偏好（15%）、收藏行为（5%）和分类多样性（5%）。

（5）实现了 JWT httpOnly Cookie 认证方案，配合 SameSite 和 Secure 属性，有效防止 XSS 和 CSRF 攻击。

（6）实现了 Docker 容器化部署，一条命令完成 MySQL、后端、前端三个服务的部署。

（7）编写了 254 个测试用例，覆盖单元测试、接口集成测试和 E2E 测试，全部通过。

### 7.2 不足与展望

（1）推荐算法优化：当前基于加权评分，未来可引入协同过滤或深度学习算法。

（2）移动端适配：未来可开发基于 uni-app 的移动应用。

（3）食物数据库扩充：当前内置 230+ 种食物，未来可进一步扩充并支持用户自定义食物。

（4）社交功能：增加家庭成员间的饮食分享和互动功能。

（5）架构优化：引入 Redis 缓存、Spring Boot Actuator 监控。
